# 미래내일 일경험 · 운영 트리아지 어드민

AX 해커톤 과제3(소셜혁신연구소) 제출 데모. 300명 참여자의 8주 출결 데이터를 바탕으로, 운영자가 **"누구에게 먼저 개입하면 수당이 지켜지는지"** 를 우선순위로 판단하도록 지원하는 운영 대시보드이다.

**라이브 데모:** https://inwoo-jang.github.io/ax-hackathon-ilgyeong-triage/
별도 설치 없이 브라우저에서 바로 실행되며, AI 문구 생성은 기본적으로 mock 모드로 동작한다. 실시간 AI 호출 방식은 아래 [실행 방법](#실행-방법)에 기술한다.

---

## 1. 문제와 접근

**현황(As-Is).** 운영자는 300명 × 8주 출결을 수기로 점검해 위험 참여자와 수당 하락 대상을 직접 판단한다. 건당 3~5분 × 300명 기준 약 15~25시간/회가 소요되며, 정산 마감 직전에 업무가 몰린다.

**접근(To-Be).** 규칙(코드)이 출석률·수당구간·개입가치를 계산해 대상을 자동 정렬하고, AI가 상태 요약·안내 문구를 생성한다. 운영자는 확인·확정·연락만 수행한다. 계산은 코드, 설명·생성은 AI, 최종 확정은 사람으로 역할을 분리한 것이 핵심 설계 원칙이다.

---

## 2. 데모 시나리오

1. **오늘 할 일** — 개입가치(지금 개입해 추가로 지킬 수 있는 수당) 높은 순으로 대상이 정렬된다.
2. 참여자 카드 선택 시 상세 패널이 열린다. **만회 분석**(현재 출석률 + 도달 가능 구간), **What-if 시뮬레이터**(남은 주차 출석 시간 입력 → 수당 실시간 재계산)를 제공한다.
3. **AI 조치 카드** — 상태 요약·우선순위 근거·문자 문구·전화 스크립트를 생성한다. "추가 지시" 입력란에 참여자 상황·톤을 지정하고 **다시 생성**을 실행하면 문구에 반영된다.
4. **연락 완료 / 출석시간 수정 / 보류**(정산 주차인 4·8주에는 **확정 완료**) 처리 후 **처리 내역**에 기록된다.

---

## 3. AI 활용 (역할 분리)

설계 원칙은 **계산·임계판정 = 규칙(코드) / 요약·생성·설명 = AI / 최종 확정 = 사람** 이다. 계산을 AI에 위임하지 않는다.

| 단계 | 담당 | 범위 |
|---|---|---|
| 출석률·수당구간·금액환산·개입가치·만회판정·What-if 재계산 | 🧮 규칙(코드) | 결정론적 영역. 정답이 정해진 계산·임계 판정 |
| ① 상태 요약 ② 우선순위 근거 ③ 안내 문구 생성(문자·전화, 단건·일괄) ④ 피드백 루프 | 🤖 AI(Claude) | 비정형 수치를 자연어로 요약·설명·생성 |
| 연락·확정·예외 판단·책임 | 🙋 사람 | 자동 확정·자동 발송 없음 |

AI가 사용되는 지점, "왜 AI인가", AI가 하지 않는 영역의 구분은 [docs/06_AI활용지점.md](docs/06_AI활용지점.md)에 상세히 정리되어 있다.

**환각 방지.** 규칙(코드)이 산출한 숫자·사실(출석률·남은 주차·필요 출석 시간·수당 구간)을 "기준 초안"으로 프롬프트에 고정하고, AI는 해당 숫자를 변경하지 않고 표현만 재작성한다. 출력은 JSON으로 강제하며 파싱 실패 시 규칙 초안으로 폴백한다.

## 4. AI 연동 — 실서비스와 데모 우회

### 실서비스
각 AI 지점은 백엔드를 통해 **Anthropic Claude API**(`POST /api/ai/recommend` 등)를 호출한다. **API 키는 서버 환경변수(`ANTHROPIC_API_KEY`)에 보관하며 프론트엔드에 노출하지 않는다.** 즉 운영 배포에는 AI API 키가 필수이다.

### 데모 우회
브라우저로 직접 실행하는 단일 HTML은 키·백엔드가 없어 API를 직접 호출할 수 없다. 데모는 아래 3경로를 두고, 상단에서 모드를 선택하며 실패 시 자동으로 mock으로 폴백한다.

| 경로 | 방식 | 키 | 비고 |
|---|---|---|---|
| ① 로컬 Claude 브리지 | `bridge.py`가 설치된 Claude Code CLI(`claude`)를 호출 | 불필요 | ⚠️ 데모/로컬 전용 우회 |
| ② Anthropic API 키 | 브라우저에서 Claude API 직접 호출(`anthropic-dangerous-direct-browser-access` 헤더) | `sk-ant-…` 런타임 입력 | ⚠️ 데모/로컬 전용 우회 |
| ③ mock(기본) | 규칙 기반 사전 생성 문구 | 불필요 | 키·인터넷 불필요. 녹화·발표 안정 |

> ①②는 데모 및 로컬 시연 전용 우회이다. 실제 서비스에서는 키를 서버에 두고 백엔드에서 호출해야 한다.

## 5. 실행 방법

### 5.1 기본 실행 (mock)
`index.html`을 브라우저로 연다. AI 문구는 규칙 기반 mock으로 생성되며 키·인터넷이 필요 없다.

### 5.2 실시간 AI — 로컬 Claude 브리지 (키 불필요)
```bash
python3 bridge.py          # http://localhost:8787 (Claude Code CLI 필요)
```
이후 `index.html` 상단의 **로컬 Claude 연결**을 선택하면 상태 표시가 🟢 로컬 Claude로 바뀌고 "다시 생성"이 실시간 호출로 동작한다.

### 5.3 실시간 AI — Anthropic API 키
`index.html` 상단 입력란에 `sk-ant-…` 키를 입력하면 🟢 실시간 AI · API로 동작한다.

### 5.4 빌드 (소스 수정 시)
```bash
python3 build.py           # src/ 를 단일 index.html 로 인라인
```

## 7. 분석 모델 요약

- **정산 구조.** 4주 단위 2회차(P1: 1~4주, P2: 5~8주). 각 기간 종료 주차(4·8주)에 정산 확정한다. 프로그램(코호트)별 현재 주차가 상이하다.
- **수당.** 직무훈련·일경험을 각각 4주 출석률 기준 구간(90% / 70-90% / 60-70% / 50-60% / 50% 미만 미지급)으로 차등 산정해 합산한다.
- **만회 판정(feasib).** `만회불가`(남은 주를 모두 채워도 미지급) / `안전`(이미 90% 이상 유지) / `출석 장려 대상`(아직 구간 상향 여지 있음 → 처리 대상). 현재 데이터 분포는 출석 장려 대상 229 / 안전 63 / 만회불가 8이다.
- **개입가치(swing).** `최대 출석 시 수당 − 현재 출석률 유지 시 수당`. 지금 개입해 추가로 지킬 수 있는 금액으로, 우선순위 큐의 기본 정렬 기준이다.
- **처리 상태.** 미처리 → 연락 완료 / 확정 완료(4·8주) / 보류.

> 모든 데이터는 더미이며 실명·실연락처가 아니다.
