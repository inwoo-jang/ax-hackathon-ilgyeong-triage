# 정보구조(IA) — 미래내일 일경험 운영 트리아지 어드민 (과제3)

> 입력: `_MODEL_BRIEF.md`(권위) · 작성: 2026-06-20 · 와이어프레임은 생략하고 데모 빌드로 바로 넘어가기 위한 최소 IA

---

## 1. 사이트맵 (데모 = SPA 단일 화면 + 탭 라우팅 + 패널/모달)

```
운영 트리아지 어드민 (SPA)
├─ [좌측 글로벌 네비 / 4메뉴] ──────────────────────
│   ├─ ① 오늘 할 일       (/)              랜딩·승부처 (우선순위 큐)
│   ├─ ② 참여자 관리       (/participants)  리스트·검색·필터·일괄 문구
│   ├─ ③ 처리 내역         (/logs)          조치 로그
│   └─ ④ 멘토별 현황       (/mentors)        멘토(프로그램)별 그룹 뷰
│
├─ [오버레이] ──────────────────────────────────────
│   ├─ 참여자 상세 패널   (우측 슬라이드/모달)  ← 큐·리스트·멘토뷰에서 진입
│   └─ 일괄 검토 모달     (모달)               ← 리스트 배치 선택 시
│
└─ [공통] 상단바: AI 모드 선택(로컬 Claude 브리지 / API 키 / mock) · 데이터 출처(더미 300명)
```

> 데모는 단일 페이지에 상태 라우팅을 얹어 좌측 탭으로 전환하는 방식을 권장한다. 상세와 일괄 검토는 라우트를 옮기지 않고 오버레이로 띄워 컨텍스트를 유지하며, 닫을 때 필터와 스크롤 위치를 그대로 보존한다.
> 상단 AI 모드는 세 가지다. 로컬 Claude 브리지는 키가 필요 없고, Anthropic API 키는 런타임에 입력받으며, mock은 키도 인터넷도 없이 녹화가 안정적이다. 호출이 실패하면 자동으로 mock으로 폴백한다.

## 2. 화면별 영역 구성 (블록 단위)

### ① 오늘 할 일 `/`
```
┌ 금액 헤드라인 (2축, 큰 숫자) ─────────────────────┐
│  오늘 개입으로 좌우되는 금액  |  서류 미비로 보류된 금액 │
│       ₩16,631,250         |      ₩4,050,000        │
├ 요약 지표 카드 4개 ──────────────────────────────┤
│  전체 300 · 출석 장려 대상 229 · 서류 미비 10 · 만회불가 8 │
├ 우선순위 큐 (메인, 개입가치 정렬) ────────────────┤
│  [정렬·필터 바]                                    │
│    정렬: 개입가치 / 출석률 / 가나다 (↑↓ 방향 토글)  │
│    필터: 지급확정 주간(4·8주) 모아보기 ·            │
│          연락처 보기 · 서류 미비만 보기              │
│  [카드] 이름·프로그램(멘토) | FeasibChip(출석장려/   │
│         안전/만회불가) | 개입가치(swing) | 한줄 근거 │
│         | 종료 D-n주 배지 | StatusChip(미처리/연락  │
│         완료/확정완료/보류)                          │
│  ↓ 기본 정렬: 개입가치(swing) desc                   │
└──────────────────────────────────────────────────┘
  카드를 클릭하면 참여자 상세 패널이 열린다.
```

### ② 참여자 관리 `/participants`
```
┌ 검색바 + 필터바 ─────────────────────────────────┐
│  [이름검색] [프로그램▾][서류상태▾][만회판정▾]        │
│  [서류 미비만 토글] [정렬▾: 개입가치/출석률/가나다]   │
├ 배치 액션바 (N개 선택 시 노출) ───────────────────┤
│  ☑ N명 선택  → [AI 일괄 문구 생성]                 │
├ 목록 테이블 ─────────────────────────────────────┤
│  ☐ 이름 | 프로그램 | JT 출석률 | WE 출석률 |        │
│    수당합산 | 개입가치 | FeasibChip | StatusChip    │
└──────────────────────────────────────────────────┘
  행을 클릭하면 상세 패널, 체크하면 일괄 검토 모달로 간다.
```

### 참여자 상세 패널 (오버레이)
```
┌ 헤더: 이름 · 프로그램 · FeasibChip · 종료 D-n주 ──┐
├ 만회 분석: JT/WE 현재 출석률 + 도달 가능 범위      │
│   (하한 min ~ 상한 max)                            │
├ 정산 기간별 현재 출석률 (1~4주 / 5~8주,            │
│   상태=정산확정 / 진행중 / 미시작)                  │
├ 수당 구간 기준표 (? 토글로 펼침)                    │
├ What-if 시뮬레이터: 주차별 남은 출석 시간 입력     │
│   출석률·수당 실시간 재계산. 기본=현 출석률 유지     │
│   시 시간, [max] 체크박스(남은 주 만땅 시뮬)        │
├ 주차별 출석 시간 (week01~08, P1/P2 구분)           │
│   + [출석시간 수정] 모드: 현 정산기간 주차만 수정,  │
│     P1은 🔒 잠금 · 저장 시 서버 전송(mock)+만회 재계산│
├ 연락 기록: 6월 캘린더 (날짜 클릭=기록 추가,         │
│   기록 클릭=수정/삭제, 처리 내역과 연동)            │
├ AI 조치 카드: 상태 요약 / 우선순위 근거 /           │
│   📱메시지 문구 / 📞전화 스크립트 /                  │
│   ✏️추가 지시(상황·톤) + [다시 생성]                │
└ 하단 고정 액션바(sticky):                          │
│   [연락 완료] [출석시간 수정] [보류]                 │
│   (4·8주 화면이면 [확정 완료] 추가)                 │
└──────────────────────────────────────────────────┘
  조치를 처리하면 처리 내역에 한 건 쌓이고, 처리분이 반영되며 큐가 갱신된다.
```

### 일괄 검토 모달 (오버레이)
```
선택한 N명에 대해 이름·근거·금액을 반영한 AI 문구 카드 N장이 뜬다.
각 카드는 인라인으로 고칠 수 있고, [일괄 발송(mock)]을 누르면 처리 내역에 N건이 남는다.
```

### ③ 처리 내역 `/logs`
```
필터: [참여자별][기간][조치유형]
테이블: 시각 | 참여자 | 조치 | 결과(확정완료 / 연락완료 / 보류)
        행을 클릭하면 상세로 이동
```

### ④ 멘토별 현황 `/mentors`
```
멘토(프로그램)별 그룹카드: 담당 N명 · 그룹 평균 출석률 ·
  출석 장려 대상 수 · 개입가치 합 하이라이트
  참여자를 클릭하면 같은 상세 패널을 공유해서 연다.
```

## 3. 네비게이션 / 진입 흐름

| From | To | 트리거 | 컨텍스트 보존 |
|---|---|---|---|
| 오늘 할 일 큐 카드 | 상세 패널 | 클릭 | 출처=queue |
| 참여자 관리 행 | 상세 패널 | 클릭 | 필터·스크롤 유지 |
| 참여자 관리 배치선택 | 일괄 검토 모달 | [AI 일괄 생성] | 선택 id 배열 |
| 상세 [연락 완료]/[보류]/[확정 완료] | 처리 내역(append) → 출처 화면 | 하단 액션바 | 큐 갱신 |
| 처리 내역 행 | 상세 패널 | 클릭 | 출처=logs |
| 멘토별 현황 참여자 | 상세 패널 | 클릭 | 출처=mentors |

## 5. 데모 구현 우선순위 (5시간 빌드 컷)

| 순위 | 범위 | 비고 |
|---|---|---|
| P0 must | 오늘 할 일(2헤드라인·요약카드·우선순위 큐) + 상세 패널(만회분석·What-if·출석시간 수정·AI카드·하단 액션바) | 시연이 도는 1루프 |
| P0 | 참여자 관리(검색·필터·정렬) | 검색 포털 |
| P1 | 일괄 검토 모달 + 처리 내역 | 스케일·이력 |
| P2 | 멘토별 현황 | 시간 여유 시만 |

> 데이터는 `participants_processed.csv` 300행을 그대로 쓴다. 출석률, 수당구간, 개입가치, 만회판정, What-if 재계산 같은 수치는 전부 코드로 처리하며 recon.py 산식을 포팅한다. AI(Claude)가 맡는 일은 상태 요약, 우선순위 근거, 단건·일괄 문구 생성, 피드백 루프뿐이고 자동 확정이나 발송은 없다.
