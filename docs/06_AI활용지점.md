# AI 활용 지점 — 어디에 연결되고, 왜 AI인가 (과제3 발표용)

> 제품: **미래내일 일경험 운영 트리아지 어드민** — 운영자가 300명 참여자의 8주 출결을 보고, 수당 구간을 지키도록 우선순위로 개입하는 대시보드.
> 설계 원칙: **계산·임계판정 = 규칙(코드) / 요약·생성·설명 = AI / 최종 확정 = 사람.** AI에 계산을 떠넘기지 않는다.
> 근거 코드: `src/app.js` (`genAI`·`aiPrompt`·`callClaude`·`callAnthropic`·`callBridge`·`checkBridge`·`regen`), `bridge.py`. 작성: 2026-06-20

---

## 0. 한 장 요약 — 역할 분리

```
 [더미 300명 · 8주 주차별 출결(직무훈련·일경험)]
          │
   🧮 규칙(코드)  ── 출석률 · 수당구간 · 금액환산 · 개입가치(swing) · 만회판정(feasib) · What-if 재계산
          │  (결정론. 정답이 있는 영역 — 숫자표를 만든다)
          ▼
   🤖 AI(Claude)  ── ① 상태 요약  ② 우선순위 근거  ③ 안내 문구 생성(문자·전화 / 단건·일괄)  ④ 피드백 루프
          │  (비정형 → 사람 말. 숫자표를 "읽고" 설명·생성만)
          ▼
   🙋 사람(운영자) ── 연락 · 확정 · 예외 · 책임. 자동 확정/발송 없음
```

> **AI는 규칙이 만든 숫자표를 읽고 설명·생성만 한다. 계산은 코드, 확정은 사람.**
> 점수를 매기는 건 코드, 납득시키는 건 AI.

## 1. AI API 연결 지점 (4곳) — 데모 화면 대응

화면은 현재 **4메뉴**(오늘 할 일 / 참여자 관리 / 처리 내역 / 멘토별 현황)와 참여자 상세의 **AI 조치 카드**(상태 요약 / 우선순위 근거 / 📱메시지 문구 / 📞전화 스크립트 / ✏️추가 지시 + 다시 생성) 기준이다.

| # | AI 기능 | 연결 화면 | 실서비스 엔드포인트 | 데모 동작 |
|---|---|---|---|---|
| ① | **상태 요약** | 참여자 상세 · AI 조치 카드(상태 요약) | `POST /api/ai/recommend` | `genAI()`가 규칙값으로 요약 초안 생성. "다시 생성" 시 실시간 호출 |
| ② | **우선순위 근거** | 오늘 할 일(큐 카드) · 상세 AI 조치 카드(우선순위 근거) | `POST /api/ai/recommend` (①과 동일 응답) | 개입가치·만회판정 근거를 자연어 한 줄로 표시 |
| ③ | **안내 문구 생성(문자·전화 / 단건·일괄)** | 상세(📱메시지·📞전화) · 참여자 관리(일괄 문구) | `POST /api/ai/recommend` (단건) / `POST /api/ai/recommend-batch` (일괄) | `aiPrompt()`로 톤·상황 지시 반영, N명 동시 생성 |
| ④ | **피드백 루프** | 처리 내역(AI 추천 ↔ 실제 대조) | `POST /api/ai/override` | 로그에 AI 초안(`ai`)·운영자 최종(`final`)·결과 누적 |

> 데모는 화면녹화 안정성을 위해 기본은 규칙 기반 **사전 생성(mock)**. "다시 생성"을 누르면 위 엔드포인트와 동일한 프롬프트로 **실시간 호출**(데모 우회 경로 사용). 동일 UI에서 실서비스 엔드포인트로 교체만 하면 라이브 전환된다. UI의 **🤖 배지**가 정확히 이 4지점에만 붙어 있어 어디가 AI인지 화면에서 바로 보인다.

## 2. 지점별 — 왜 AI를 쓰는가 (규칙으로 안 되는 비정형 언어화)

### ① 상태 요약 🤖
- **하는 일**: 주차별 출석시간·출석률·수당구간·만회판정 값 → "현재 6주차(종료까지 2주), 출석률 직무훈련 78%·일경험 64%, 남은 2주 직무훈련 6h·일경험 8h 더 채우면 다음 구간 도달" 한 문단.
- **왜 AI?** 정형 수치를 **맥락 있는 자연어 한 문단으로 압축**하는 건 비정형 언어화. 규칙은 "64%"까지만 만들고, "그래서 지금 무엇을 해야 하는지"의 서술은 못 만든다.

### ② 우선순위 근거 🤖
- **하는 일**: "남은 출석으로 수당 구간이 바뀌는 출석 장려 대상 → 개입가치 ○○원. 종료까지 2주라 지금 독려해야 만회 가능" 같은 납득 가능한 이유 문장.
- **왜 AI?** 개입가치(swing)·만회판정(feasib)은 규칙이 매기지만, **운영자가 신뢰하고 판단을 표준화**하려면 그 숫자가 "왜 먼저 봐야 하는지"로 말이 풀려야 한다. 설명 레이어 = AI.

### ③ 안내 문구 생성 (문자·전화 / 단건·일괄) 🤖 ★스케일
- **하는 일**: 대상별 맥락(이름·남은 주차·필요 출석시간·수당 영향·서류 상태)에 맞춘 문자(msg)·전화 스크립트(call). 톤 지시(따뜻/정중/간결)와 상황 지시를 반영. 일괄 선택 시 N명 동시.
- **왜 AI?** 300명에게 **각자 다른 맞춤 문구**를 사람이 쓰면 비현실적(건당 수 분 × 수백). 생성은 AI가 스케일 처리하고 사람은 검토·발송만. → "단건 반복을 AI 스케일로".

### ④ 피드백 루프 🤖
- **하는 일**: 운영자가 수정한 최종 문구·반려 사유를 처리 내역에 누적(AI 초안 ↔ 실제 대조) → 추천 적중률 사후 검증·프롬프트 개선.
- **왜 AI?** **휴먼인더루프 데이터로 생성 품질을 운영 중 개선**. AI 추천 신뢰도를 시간이 갈수록 끌어올리는 학습 고리.

## 3. AI가 **안 하는** 것 (의도적 분리 — 전부 규칙/코드)

| 영역 | 담당 | 왜 AI가 아닌가 |
|---|---|---|
| 출석률 계산(Σ출석 ÷ Σ배정, 4주 구간) | 🧮 규칙 | 결정론. AI 쓰면 오히려 틀리고 느림 |
| 수당 구간 판정(90% / 70-90% / 60-70% / 50-60% / <50%) | 🧮 규칙 | 규정표 = 정답. 임계 판정은 코드가 정확 |
| 금액 환산(직무훈련·일경험 별개 계산 후 합산) | 🧮 규칙 | 수당표 매핑. 산출값을 AI가 읽기만 |
| 개입가치(swing) 산출 = max(0, 최대출석시 수당 − 유지시 수당) | 🧮 규칙 | 정렬 기준 숫자. 결정론적 계산 |
| 만회판정(feasib): 만회불가 / 안전 / 출석 장려 대상 | 🧮 규칙 | max·cur 임계 비교. 코드가 판정 |
| What-if 실시간 재계산(주차별 입력 → 출석률·수당) | 🧮 규칙 | 운영자가 보는 값과 동일 산식 재사용(일관성) |
| 최종 조치 확정·발송 | 🙋 사람 | 책임·예외 판단. AI 자동확정·자동발송 금지 |

> 이 경계가 선명할수록 "AI 활용 방식" 채점이 오른다. **모든 걸 AI에 떠넘기지 않은 것**이 핵심 설계 포인트.

## 4. 실서비스 vs 데모 우회

### 실서비스 (운영 배포)
- 백엔드가 **Anthropic Claude API**를 호출. 프론트는 `POST /api/ai/recommend`(단건)·`/api/ai/recommend-batch`(일괄)·`/api/ai/override`(피드백)만 호출.
- **API 키는 서버 환경변수 `ANTHROPIC_API_KEY`에 보관, 프론트에 미노출.** → AI API 키 필수.

### 데모 우회 (3경로 · 자동 폴백)
화면녹화·발표장 시연용. 우상단에서 모드 선택, 실패 시 자동으로 mock 폴백(`regen()`).

| 경로 | 방식 | 키 | 비고 |
|---|---|---|---|
| ① 로컬 Claude 브리지 | `bridge.py`가 설치·로그인된 Claude Code CLI(`claude -p`)를 호출 (`callBridge` → `http://localhost:8787`) | **불필요** (기존 Claude Code 인증) | ⚠️ **데모 전용 우회** |
| ② Anthropic 키 런타임 입력 | 브라우저가 `api.anthropic.com`에 직접 호출 (`callAnthropic`), `anthropic-dangerous-direct-browser-access: true` 헤더, 모델 `claude-sonnet-4-6` | 런타임 입력 | ⚠️ **데모 전용 우회**(브라우저 직접 호출) |
| ③ mock | `genAI()` 규칙 기반 사전 생성 | **불필요** | 키·인터넷 불필요 · 녹화 안정 |

> ⚠️ ①②는 **데모/로컬 시연 전용 우회**다. 실서비스에서는 키를 서버에 두고 백엔드가 호출한다(브라우저 직접 호출·CLI 호출 금지).
> 호출 우선순위(`regen`/`callClaude`): API 키가 있으면 ① 직접 호출 → 없으면 ② 브리지 → 둘 다 실패 시 ③ mock.

## 5. 프롬프트 설계 + 환각 방지

**핵심**: 규칙이 만든 숫자·사실(출석률·남은 주차·더 채울 시간·수당 구간)을 **"기준 초안"으로 system에 고정**하고, AI는 **표현만 재작성**한다. 숫자 변경 금지. 실제 `aiPrompt()` 방식 그대로:

```js
// src/app.js — aiPrompt(p, instruction)
const g = genAI(p); // ← 규칙이 만든 사실 초안(숫자·사실)

const sys =
"너는 '미래내일 일경험' 사업의 운영 담당자다. 아래 '기준 초안'에 담긴 숫자와 사실" +
"(출석률, 남은 주차, 더 채울 시간, 수당 구간)은 절대 바꾸지 마라. 운영자의 추가 지시" +
"(참여자 상황·톤)를 자연스럽게 반영해 문자 안내(msg)와 전화 통화 스크립트(call)를 다시 써라. " +
"전화 스크립트는 ①인사 ②현황 ③핵심(필요 시간·구간) ④경청 ⑤마무리 흐름을 유지한다. " +
"정중하고 따뜻한 운영자 말투. 반드시 JSON만 출력: {\"msg\":\"...\",\"call\":\"...\"}";

const user =
`[기준 초안 · 문자]\n${g.msg}\n\n` +
`[기준 초안 · 전화]\n${g.call}\n\n` +
`[운영자 추가 지시]\n${instruction || '(없음 — 기본 톤 유지)'}\n\n` +
`위 지시를 반영하되 숫자·사실은 그대로 두고 msg와 call을 JSON으로 다시 작성해줘.`;
```

**환각 방지 장치**
- 숫자·사실은 코드(`genAI`)가 만든 초안으로 system·user에 **이미 박아 넣음** → AI는 새 숫자를 만들 입력이 없다.
- 출력은 **JSON 강제**(`{msg, call}`), 파싱 실패 시 규칙 초안(`g.msg`·`g.call`)으로 폴백(`parseAI`).
- 상태 요약(summary)·우선순위 근거(reason)는 규칙 초안을 **그대로 유지**, AI는 문자·전화 표현만 재작성.

## 6. 발표 멘트 (30초)

> "우리는 AI를 아무 데나가 아니라 **딱 4곳**에 씁니다. 계산은 전부 코드가 합니다 — 출석률, 수당 구간, 개입가치, 만회 판정, What-if는 정답이 있으니까요. AI는 그 숫자표를 **읽고**, ① 상태를 요약하고 ② 왜 먼저 봐야 하는지 근거를 설명하고 ③ 300명에게 문자·전화 맞춤 문구를 단건·일괄로 생성하고 ④ 운영자 피드백으로 스스로 개선합니다. 숫자는 system에 고정해 **AI가 바꾸지 못하게** 했고, 화면의 🤖 배지가 붙은 곳이 정확히 AI가 일하는 지점입니다. 실서비스는 키를 서버에 두고 백엔드가 Claude를 호출하고, 최종 확정은 항상 사람입니다."
