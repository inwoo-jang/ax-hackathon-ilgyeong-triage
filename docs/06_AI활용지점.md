# AI 활용 지점: 어디에 연결되고, 왜 AI인가 (과제3 발표용)

> 제품: 미래내일 일경험 운영 트리아지 어드민. 운영자가 참여자 300명의 8주 출결을 보면서, 수당 구간을 지키도록 우선순위를 매겨 개입하는 대시보드다.
> 설계 원칙은 단순하다. 계산과 임계판정은 규칙(코드)이 맡고, 요약·생성·설명은 AI가, 최종 확정은 사람이 한다. 계산을 AI에 떠넘기지 않는다는 뜻이다.
> 근거 코드: `src/app.js` (`genAI`·`aiPrompt`·`callClaude`·`callAnthropic`·`callBridge`·`checkBridge`·`regen`), `bridge.py`. 작성: 2026-06-20

---

## 0. 한 장 요약: 역할 분리

```
 [더미 300명 · 8주 주차별 출결(직무훈련·일경험)]
          │
   🧮 규칙(코드)  ── 출석률 · 수당구간 · 금액환산 · 개입가치(swing) · 만회판정(feasib) · What-if 재계산
          │  (결정론. 정답이 있는 영역 — 숫자표를 만든다)
          ▼
   🤖 AI(Claude)  ── ① 상태 요약  ② 우선순위 근거  ③ 안내 문구 생성(문자·전화 / 단건·일괄)  ④ 피드백 루프
          │  (비정형 → 사람 말. 숫자표를 "읽고" 설명·생성만)
          ▼
   🙋 사람(운영자) ── 연락 · 확정 · 예외 · 책임. 자동 확정/발송 없음
```

> AI는 규칙이 만들어 놓은 숫자표를 읽어서 설명하고 문구를 생성할 뿐이다. 계산은 코드가, 확정은 사람이 한다.
> 점수를 매기는 건 코드의 몫이고, 그 점수를 운영자에게 납득시키는 게 AI의 몫이다.

## 1. AI API 연결 지점 (4곳)과 데모 화면 대응

지금 화면은 4메뉴(오늘 할 일 / 참여자 관리 / 처리 내역 / 멘토별 현황)와, 참여자 상세에 붙는 AI 조치 카드(상태 요약 / 우선순위 근거 / 📱메시지 문구 / 📞전화 스크립트 / ✏️추가 지시 + 다시 생성)를 기준으로 잡았다.

| # | AI 기능 | 연결 화면 | 실서비스 엔드포인트 | 데모 동작 |
|---|---|---|---|---|
| ① | 상태 요약 | 참여자 상세 · AI 조치 카드(상태 요약) | `POST /api/ai/recommend` | `genAI()`가 규칙값으로 요약 초안 생성. "다시 생성" 시 실시간 호출 |
| ② | 우선순위 근거 | 오늘 할 일(큐 카드) · 상세 AI 조치 카드(우선순위 근거) | `POST /api/ai/recommend` (①과 동일 응답) | 개입가치·만회판정 근거를 자연어 한 줄로 표시 |
| ③ | 안내 문구 생성(문자·전화 / 단건·일괄) | 상세(📱메시지·📞전화) · 참여자 관리(일괄 문구) | `POST /api/ai/recommend` (단건) / `POST /api/ai/recommend-batch` (일괄) | `aiPrompt()`로 톤·상황 지시 반영, N명 동시 생성 |
| ④ | 피드백 루프 | 처리 내역(AI 추천 ↔ 실제 대조) | `POST /api/ai/override` | 로그에 AI 초안(`ai`)·운영자 최종(`final`)·결과 누적 |

> 발표장 화면녹화가 끊기지 않도록, 데모 기본값은 규칙 기반 사전 생성(mock)으로 둔다. "다시 생성"을 누르면 위 엔드포인트와 똑같은 프롬프트로 실시간 호출이 나가는데, 이때는 데모 우회 경로를 탄다. UI는 그대로 두고 실서비스 엔드포인트로만 바꿔 끼우면 바로 라이브로 넘어간다. 그리고 UI의 🤖 배지는 딱 이 4지점에만 달아 놨기 때문에, 어디가 AI인지 화면만 봐도 구분된다.

## 2. 지점별로, 왜 굳이 AI인가 (규칙으로는 안 되는 비정형 언어화)

### ① 상태 요약 🤖
주차별 출석시간·출석률·수당구간·만회판정 값을 받아서 "현재 6주차(종료까지 2주), 출석률 직무훈련 78%·일경험 64%, 남은 2주 직무훈련 6h·일경험 8h 더 채우면 다음 구간 도달" 같은 한 문단으로 묶어낸다.
정형 수치를 맥락 있는 자연어 한 문단으로 압축하는 일은 결국 비정형 언어화라서 AI가 맞다. 규칙은 "64%"라는 숫자까지만 만들 뿐, "그래서 지금 뭘 해야 하는지"를 문장으로 풀어내지는 못한다.

### ② 우선순위 근거 🤖
"남은 출석으로 수당 구간이 바뀌는 출석 장려 대상, 개입가치 ○○원. 종료까지 2주라 지금 독려해야 만회 가능" 처럼 운영자가 납득할 수 있는 이유 문장을 만든다.
개입가치(swing)와 만회판정(feasib)을 매기는 건 규칙이지만, 운영자가 그 숫자를 믿고 판단을 표준화하려면 "왜 이걸 먼저 봐야 하는지"가 말로 풀려야 한다. 그 설명 레이어를 AI가 담당한다.

### ③ 안내 문구 생성 (문자·전화 / 단건·일괄) 🤖
대상마다 다른 맥락(이름·남은 주차·필요 출석시간·수당 영향·서류 상태)에 맞춰 문자(msg)와 전화 스크립트(call)를 쓴다. 톤 지시(따뜻/정중/간결)와 상황 지시를 반영하고, 일괄을 고르면 N명을 한 번에 처리한다.
300명에게 각자 다른 맞춤 문구를 사람이 직접 쓰는 건 현실적으로 불가능하다. 건당 몇 분씩 수백 건이다. 그래서 생성은 AI가 스케일로 처리하고, 사람은 검토하고 발송만 한다. 단건 반복을 AI 스케일로 바꾸는 지점이라 4곳 중에서도 효과가 가장 크다.

### ④ 피드백 루프 🤖
운영자가 고친 최종 문구와 반려 사유를 처리 내역에 쌓아 AI 초안과 실제를 대조하고, 이걸로 추천 적중률을 사후 검증하면서 프롬프트를 개선한다.
휴먼인더루프 데이터로 생성 품질을 운영 중에 끌어올리는 구조다. 시간이 갈수록 AI 추천의 신뢰도가 올라가는 학습 고리인 셈이다.

## 3. AI가 안 하는 것 (의도적으로 떼어 둔 영역, 전부 규칙/코드)

| 영역 | 담당 | 왜 AI가 아닌가 |
|---|---|---|
| 출석률 계산(Σ출석 ÷ Σ배정, 4주 구간) | 🧮 규칙 | 결정론. AI 쓰면 오히려 틀리고 느림 |
| 수당 구간 판정(90% / 70-90% / 60-70% / 50-60% / <50%) | 🧮 규칙 | 규정표 = 정답. 임계 판정은 코드가 정확 |
| 금액 환산(직무훈련·일경험 별개 계산 후 합산) | 🧮 규칙 | 수당표 매핑. 산출값을 AI가 읽기만 |
| 개입가치(swing) 산출 = max(0, 최대출석시 수당 − 유지시 수당) | 🧮 규칙 | 정렬 기준 숫자. 결정론적 계산 |
| 만회판정(feasib): 만회불가 / 안전 / 출석 장려 대상 | 🧮 규칙 | max·cur 임계 비교. 코드가 판정 |
| What-if 실시간 재계산(주차별 입력 → 출석률·수당) | 🧮 규칙 | 운영자가 보는 값과 동일 산식 재사용(일관성) |
| 최종 조치 확정·발송 | 🙋 사람 | 책임·예외 판단. AI 자동확정·자동발송 금지 |

> 이 경계가 선명할수록 "AI 활용 방식" 채점이 좋아진다. 모든 걸 AI에 떠넘기지 않았다는 점이 이 설계의 핵심이다.

## 4. 실서비스 vs 데모 우회

### 실서비스 (운영 배포)
운영 환경에서는 백엔드가 Anthropic Claude API를 호출하고, 프론트는 `POST /api/ai/recommend`(단건), `/api/ai/recommend-batch`(일괄), `/api/ai/override`(피드백) 세 엔드포인트만 부른다. API 키는 서버 환경변수 `ANTHROPIC_API_KEY`에 두고 프론트에는 절대 노출하지 않는다. 즉 운영에서는 AI API 키가 필수다.

### 데모 우회 (3경로, 자동 폴백)
화면녹화나 발표장 시연을 위한 경로다. 우상단에서 모드를 고르고, 실패하면 알아서 mock으로 떨어진다(`regen()`).

| 경로 | 방식 | 키 | 비고 |
|---|---|---|---|
| ① 로컬 Claude 브리지 | `bridge.py`가 설치·로그인된 Claude Code CLI(`claude -p`)를 호출 (`callBridge` → `http://localhost:8787`) | 불필요 (기존 Claude Code 인증) | ⚠️ 데모 전용 우회 |
| ② Anthropic 키 런타임 입력 | 브라우저가 `api.anthropic.com`에 직접 호출 (`callAnthropic`), `anthropic-dangerous-direct-browser-access: true` 헤더, 모델 `claude-sonnet-4-6` | 런타임 입력 | ⚠️ 데모 전용 우회(브라우저 직접 호출) |
| ③ mock | `genAI()` 규칙 기반 사전 생성 | 불필요 | 키·인터넷 불필요 · 녹화 안정 |

> ⚠️ ①과 ②는 어디까지나 데모와 로컬 시연 전용 우회다. 실서비스에서는 키를 서버에 두고 백엔드가 호출하며, 브라우저 직접 호출이나 CLI 호출은 쓰지 않는다.
> 호출 우선순위(`regen`/`callClaude`)는 이렇다. API 키가 있으면 ①로 직접 호출하고, 없으면 ② 브리지를 타며, 둘 다 안 되면 ③ mock으로 폴백한다.

## 5. 프롬프트 설계 + 환각 방지

기본 아이디어는 이렇다. 규칙이 만든 숫자와 사실(출석률·남은 주차·더 채울 시간·수당 구간)을 "기준 초안"으로 system에 박아 두고, AI는 표현만 다시 쓰게 한다. 숫자는 손대지 못한다. 실제 `aiPrompt()`가 쓰는 방식 그대로 옮기면 다음과 같다.

```js
// src/app.js — aiPrompt(p, instruction)
const g = genAI(p); // ← 규칙이 만든 사실 초안(숫자·사실)

const sys =
"너는 '미래내일 일경험' 사업의 운영 담당자다. 아래 '기준 초안'에 담긴 숫자와 사실" +
"(출석률, 남은 주차, 더 채울 시간, 수당 구간)은 절대 바꾸지 마라. 운영자의 추가 지시" +
"(참여자 상황·톤)를 자연스럽게 반영해 문자 안내(msg)와 전화 통화 스크립트(call)를 다시 써라. " +
"전화 스크립트는 ①인사 ②현황 ③핵심(필요 시간·구간) ④경청 ⑤마무리 흐름을 유지한다. " +
"정중하고 따뜻한 운영자 말투. 반드시 JSON만 출력: {\"msg\":\"...\",\"call\":\"...\"}";

const user =
`[기준 초안 · 문자]\n${g.msg}\n\n` +
`[기준 초안 · 전화]\n${g.call}\n\n` +
`[운영자 추가 지시]\n${instruction || '(없음 — 기본 톤 유지)'}\n\n` +
`위 지시를 반영하되 숫자·사실은 그대로 두고 msg와 call을 JSON으로 다시 작성해줘.`;
```

환각을 막는 장치는 세 겹이다. 우선 숫자와 사실은 코드(`genAI`)가 만든 초안 형태로 system과 user에 이미 들어가 있으니, AI 입장에서는 새 숫자를 지어낼 입력 자체가 없다. 출력은 `{msg, call}` JSON으로 강제하고, 파싱이 실패하면 규칙 초안(`g.msg`·`g.call`)으로 폴백한다(`parseAI`). 그리고 상태 요약(summary)과 우선순위 근거(reason)는 규칙 초안을 그대로 쓰며, AI가 다시 쓰는 건 문자·전화 표현뿐이다.

## 6. 발표 멘트 (30초)

> "우리는 AI를 아무 데나 쓰지 않고 딱 4곳에만 씁니다. 계산은 전부 코드가 합니다. 출석률, 수당 구간, 개입가치, 만회 판정, What-if는 정답이 있으니까요. AI는 그 숫자표를 읽어서, 상태를 요약하고, 왜 먼저 봐야 하는지 근거를 설명하고, 300명에게 문자·전화 맞춤 문구를 단건이든 일괄이든 생성하고, 운영자 피드백으로 스스로 개선합니다. 숫자는 system에 고정해서 AI가 바꾸지 못하게 막았고, 화면에 🤖 배지가 붙은 자리가 정확히 AI가 일하는 지점입니다. 실서비스에서는 키를 서버에 두고 백엔드가 Claude를 호출하고, 최종 확정은 언제나 사람이 합니다."
