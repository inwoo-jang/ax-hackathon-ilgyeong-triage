## Step 1. 데이터 형태

데이터는 `src/data.json`에 JSON 배열로 들어 있고, 참여자 한 명이 레코드 하나에 대응한다. 규모는 300명이다.

| 항목 | 내용 |
|---|---|
| 형식 | JSON 배열 (`src/data.json`) |
| 규모 | 300명 (참여자 1명 = 레코드 1개) |
| 형태 | 참여자 단위. 주차별 출석은 8칸 배열로 내장 (`jt_w/jt_wt`, `we_w/we_wt`) |
| 시계열 | 8주 = 4주 단위 2회차 (P1: 1~4주 / P2: 5~8주) |
| 카테고리 | `pg`(프로그램) 10종: 미래내일 일경험_A~E / 직무훈련_A~E, 각 30명씩 균등 |
| 개인정보 | `nm`(이름·실명 아님·마스킹 안 함), `gd`(성별), `by`(생년월일), `doc`(서류상태) |

핵심 필드는 다음과 같다. `nm` 이름, `gd` 성별, `by` 생년월일, `pg` 프로그램, `doc` 서류상태가 참여자 속성이고, 출석은 `jt_w[8]/jt_wt[8]`(직무훈련 주차별 [출석 h / 배정 h])와 `we_w[8]/we_wt[8]`(일경험 주차별 [출석 h / 배정 h]) 두 묶음으로 담긴다.

## Step 2. 스키마 + 품질 체크

| 컬럼 그룹 | 타입 | 품질 |
|---|---|---|
| `pg` (프로그램) | 범주(10종) | 균등 30명씩. 수당종류와는 무관하다(아래 주의 참고) |
| `nm` (이름) | 문자 | 실명 아님. 동명이인이 있을 수 있어 이름만으로 식별하면 안 된다 |
| `gd` (성별) | 범주 | 여 150 / 남 150 균등 |
| `doc` (서류상태) | 범주 | 완료 / 미비 |
| `jt_w/jt_wt`, `we_w/we_wt` | 정수 배열[8] (시간) | 결측·이상치·논리오류(출석>배정) 0건. 깨끗한 더미 |

품질 자체는 깨끗해서 별도 클리닝은 필요 없다. 다만 분모를 고정하면 안 된다. 주차별 배정시간(`*_wt`)이 직무훈련 10~12h, 일경험 8~10h로 매주 달라지기 때문이다. 출석률은 늘 4주 구간의 `Σ출석 ÷ Σ배정`으로 계산한다.

## Step 3. 코호트(프로그램)별 현재 주차

프로그램마다 운영 진도가 달라서 현재 주차(cw)가 코호트별로 제각각이다. 매주 한 주씩 이력이 쌓이는 구조라, 이미 확정된 출석률(지난 주차)과 아직 만회 여지가 남은 구간(남은 주차)이 코호트마다 다르게 나뉜다.

| 현재 주차 | 인원 | 해당 코호트 |
|---|---|---|
| 2주차 | 60 | 미래내일 일경험_A · 직무훈련_C |
| 3주차 | 30 | 직무훈련_A |
| 4주차 | 30 | 미래내일 일경험_D |
| 5주차 | 60 | 미래내일 일경험_B · 직무훈련_D |
| 6주차 | 30 | 직무훈련_B |
| 7주차 | 30 | 미래내일 일경험_E |
| 8주차 | 60 | 미래내일 일경험_C · 직무훈련_E |

> 5주차 이상(P2 단계) 코호트는 P1(1~4주)이 이미 정산 확정된 상태라 서류 미비가 없다. 만회 판단은 현재 진행 중인 기간만 대상으로 본다.

## Step 4. 출석 시간 구조 + 정산

수당은 직무훈련(`jt`)과 일경험(`we`) 두 종류이고, 각각 따로 계산한 뒤 합산한다. program_name은 수당 종류와 무관하다는 점에 주의한다.

주차별 데이터는 [출석 h / 배정 h] 형태로 그대로 들고 있다. 배정시간은 직무훈련 10~12h, 일경험 8~10h 범위에서 주차마다 달라진다.

정산은 4주 단위로 두 번 끊는다. P1이 1~4주, P2가 5~8주이고, 각 기간이 끝나는 4주차·8주차 시점에 출석률이 확정된다. 4주 구간 출석률은 구간 합으로 본 `Σ출석 ÷ Σ배정`이며, 직무훈련과 일경험을 따로 구한다.

## Step 5. 규칙 계산 (결정론 — 코드로 완결, AI 아님)

수당표는 브리프 값을 그대로 따른다.

- 직무훈련: 90%+ 150,000 / 70-90% 131,250 / 60-70% 112,500 / 50-60% 93,750 / <50% 0
- 일경험: 90%+ 300,000 / 70-90% 262,500 / 60-70% 225,000 / 50-60% 187,500 / <50% 0

현재 기간을 기준으로 `kcalc`는 세 값을 잡는다. cur은 지난 주차까지 확정된 출석률, max는 남은 주를 다 채웠을 때의 상한, min은 남은 주를 하나도 못 채웠을 때의 하한이다.

만회 판정(`feasib`)은 세 갈래다. 직무·일경험 둘 다 max가 50% 미만이면 만회불가, 둘 다 cur이 90% 이상이면 안전, 나머지는 출석 장려 대상으로 둔다.

개입가치(swing)는 `max(0, 최대출석시 수당 − 현재 출석률 유지시 수당)`으로 계산하고, cur이 없으면 직전 P1 출석률로 대체한다. 큐의 기본 정렬키가 이 값이다. 처리 대상(actionable)은 출석 장려 대상이거나, 서류 미비이면서 만회가 가능한 경우를 묶는다.

## Step 7. 인사이트 3줄 (발표용)

1. 300명 가운데 229명이 출석 장려 대상이다. 남은 출석을 채우면 수당 구간이 바뀌는 사람들로, 출석률을 단순 %가 아니라 수당 경계까지 얼마나 여유가 있는지로 보면 개입으로 지급액을 방어할 여력이 232명에게 걸려 있다. 트리아지가 왜 필요한지를 데이터가 보여준다.
2. 오늘 개입으로 좌우되는 금액은 ₩16,631,250이다. 운영자가 출석 장려 대상을 개입가치(swing) 큰 순으로 처리하면 이 금액이 그대로 지급 방어로 이어진다. 큐 맨 위에 가장 큰 돈이 걸린 사람이 오게 된다.
3. 서류 미비 10명과 보류 금액 ₩4,050,000은 출석과 무관한 행정 블로커다. 출석이 양호해도 서류가 완비되기 전까지는 지급되지 않으므로, 별도 헤드라인으로 떼어 "서류 독촉" 트랙으로 처리한다.

## 발표/구현 시 주의

1. program_name으로 수당 종류를 가르면 안 된다. `직무훈련_A` 참여자에게도 일경험 시간이, `미래내일 일경험_A` 참여자에게도 직무훈련 시간이 채워져 있다. 두 수당을 각각 계산해 합산하는 구조다.
2. 두 수당은 따로 합산한다. 직무훈련(`jt`)과 일경험(`we`)을 각각 출석률, 구간, 금액 순으로 산정한 뒤 더한다. 한쪽이 미지급이어도 다른 쪽은 지급될 수 있다.
3. 주차별 배정시간이 변하므로 분모를 고정하면 안 된다. 배정시간(`*_wt`)이 매주 다르기 때문에 출석률은 반드시 4주 구간의 `Σ출석 ÷ Σ배정`으로 계산한다. "주당 N시간" 같은 고정값으로 나누면 틀린다.
4. 이름만으로 식별하지 않는다. `nm`은 실명이 아니고 동명이인이 있을 수 있어, 식별은 레코드 단위 `id`로 한다.
5. 코호트별로 현재 주차가 다르다. cw가 2~8주에 분산돼 있고, P2 단계(5주 이상) 코호트는 P1 서류가 완비된 상태이므로 만회 판단은 현재 진행 기간만 본다.
